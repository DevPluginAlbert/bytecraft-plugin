# BuiltByBit Listing Templates & Asset Pipeline

Reusable templates for every BuiltByBit plugin listing. Copy the templates below, fill in the bracketed fields, and ship.

## Directory Structure

```
docs/builtbybit-listing-templates/
├── builtbybit-listing-templates.md   ← this file (master template)
├── screenshot-spec.md                ← screenshot specs and naming
├── feature-list-format.md            ← feature list template
├── changelog-format.md               ← changelog/version format
├── faq-format.md                     ← FAQ section template
└── versioning-conventions.md         ← versioning and release rules
```

---

## Quick Reference: What Every Listing Needs

1. **Screenshot assets** (6-8 images, 16:9, see [screenshot-spec.md](screenshot-spec.md))
2. **Feature list** (see [feature-list-format.md](feature-list-format.md))
3. **Changelog** for current version (see [changelog-format.md](changelog-format.md))
4. **FAQ section** (see [faq-format.md](faq-format.md))
5. **Version number** matching `plugin.yml` (see [versioning-conventions.md](versioning-conventions.md))

## Listing Description Template

Copy this block into the BuiltByBit listing description editor:

```
[Plugin Name] — [one-line value proposition, e.g. "Automated server backups with Discord alerts"]

---

## About

[2-3 sentence description. What problem does it solve? Who is it for? Why is it better than alternatives?]

---

## Features

[Use the feature list template — see below]

---

## Screenshots

[Embed 6-8 screenshots following the screenshot spec]

---

## Changelog

[Use the changelog format — see below]

---

## FAQ

[Use the FAQ format — see below]

---

## Support & Feedback

```
